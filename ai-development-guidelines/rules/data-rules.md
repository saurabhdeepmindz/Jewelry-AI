# Data Rules

## Database Rules

- **Soft deletes only** — all entities have `is_deleted` flag; hard deletes forbidden
- **Audit columns required** — `created_at`, `updated_at`, `created_by` on all tables
- **UUIDs as primary keys** — never integer auto-increment PKs
- **Parameterized queries only** — never format SQL strings with user input
- **No raw SQL in services** — all DB access through SQLAlchemy ORM or Repository
- **Async only** — all DB operations use `AsyncSession` from asyncpg

## Migrations (Alembic)

- Every schema change requires a new migration file
- Migrations MUST be reviewed before applying to any environment
- Migration files are committed to git
- Never modify an applied migration — create a new one to fix it
- Run `alembic upgrade head` on startup (via `scripts/run_migrations.sh`)
- Rollback script required for every destructive migration

## Connection Pooling

- Min pool size: 5
- Max pool size: 20
- Timeout: 30 seconds
- Configured in `src/db/session.py`

## Data Validation

- All data validated by Pydantic models before DB write
- DB constraints (NOT NULL, UNIQUE) are the last line of defense — not the first
- Validate external API responses with Pydantic models before using

## Vector Store (pgvector)

- Lead embeddings stored in `leads.embedding` (VECTOR(1536))
- Used for semantic deduplication on ingest
- Index type: IVFFlat with `lists=100` for performance
- Embeddings generated by `text-embedding-3-small` (OpenAI)
