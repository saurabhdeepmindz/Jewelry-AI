Annotations — WF-001: Login

  • [A1] username field → POST /api/v1/auth/login body field username
  • [A2] password field → POST /api/v1/auth/login body field password. Never stored in localStorage or session state beyond login.
  • [A3] Error toast message is always "Invalid username or password" — no distinction between wrong username and wrong password (security rule US-039 AC3).
  • [A4] On 200 response, store access_token and refresh_token in st.session_state. Redirect to WF-002.
  • [A5] role returned in login response → stored as st.session_state["role"] → used for role-based screen visibility.
  • [A6] Token refresh happens silently before expiry. No UI for refresh — handled by API client interceptor.
