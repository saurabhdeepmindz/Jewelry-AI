Annotations — WF-005: Analytics Funnel

  • [A1] Filters → GET /api/v1/analytics/funnel?from=2026-03-01&to=2026-03-19&source=&rep_id=
  • [A2] KPI row: funnel_data.ingested, .matched, .enriched, .scored, .eligible, .outreached, .replied
  • [A3] Funnel chart: Plotly go.Funnel in production. Grayscale horizontal bars in wireframe. Stage counts labelled on bars. Drop percentage shown between stages.
  • [A4] Leads over time: Plotly go.Scatter (line chart). x-axis = date, y-axis = leads created. Aggregated by day.
  • [A5] Source quality table: computed from lead data grouped by Lead.source. Columns: matched %, eligible %, reply rate %, avg score.
  • [A6] Source values: gmt, trade_book, rapid_list, manual, api (from LeadSource enum)
  • [A7] Empty state: when funnel_data.ingested == 0 for selected date range — show empty state, hide charts and table
  • [A8] Data freshness: no cache on analytics endpoints (US-035 AC3). Every page load fetches fresh data from DB.
  • [A9] Role filtering: rep → ?rep_id={current_user_id} added server-side. Manager/admin → no rep_id filter unless explicitly selected.
  • [A10] Representative filter dropdown: only visible to manager/admin. Hidden for rep (rep always sees own data only).
