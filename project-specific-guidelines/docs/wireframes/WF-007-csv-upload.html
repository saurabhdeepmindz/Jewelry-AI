  • [A2] File info: show filename + estimated row count (count CSV newlines client-side) after file selected.

  • [A4] Upload button: disabled until file selected. Enabled once file chosen.
  • [A5] Column spec: company_name (required), source (required, enum), domain (optional), country (optional).
  • [A6] job_id: UUID returned in 202 response body. Display as reference for support queries.
  • [A7] Progress bar: animates while status == "processing". No real % — use indeterminate animation.
  • [A8] Row progress: show created + skipped_duplicates + skipped_invalid as approximate "processed" count if available in intermediate polling.
  • [A9] Polling: GET /api/v1/leads/jobs/{job_id} every 2s. Max wait: 5 minutes. Show timeout error if still processing after 5 min.
  • [A10] Results: from final GET /api/v1/leads/jobs/{job_id} when status == "completed". Fields: created, skipped_duplicates, skipped_invalid, errors.
  • [A11] Result annotations: explain why rows were skipped (duplicate domain / invalid source or company_name). Static explanatory text.
